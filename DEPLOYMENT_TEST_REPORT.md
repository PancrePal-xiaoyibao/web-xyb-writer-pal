## ⚠️ 已识别的潜在问题和解决方案

### 问题 1: Docker 构建失败

**可能原因**:
- 网络连接不稳定
- 磁盘空间不足
- Docker 镜像拉取超时

**解决方案**:
```bash
# 1. 检查网络
ping 8.8.8.8

# 2. 检查磁盘
df -h

# 3. 清理 Docker 缓存
docker system prune -a

# 4. 重试构建
docker compose build --no-cache
```

### 问题 2: 数据库连接失败

**可能原因**:
- DATABASE_URL 配置错误
- Supabase 连接池问题
- 防火墙阻止

**解决方案**:
```bash
# 1. 测试连接
psql "$DATABASE_URL"

# 2. 检查环境变量
grep DATABASE_URL .env

# 3. 查看 Compose 日志
docker compose logs app
```

### 问题 3: LLM API 调用失败

**可能原因**:
- API Key 无效或过期
- API 配额用尽
- 网络超时

**解决方案**:
```bash
# 1. 验证 API Key
curl -X POST https://api.stepfun.com/v1/chat/completions \
  -H "Authorization: Bearer $STEPFUN_API_KEY"

# 2. 检查 .env 配置
grep STEPFUN_API_KEY .env

# 3. 查看应用日志
docker compose logs app | grep -i "llm\|error"
```

### 问题 4: 磁盘空间不足

**可能原因**:
- Docker 镜像过大
- 日志文件堆积
- 输出文件过多

**解决方案**:
```bash
# 1. 检查磁盘使用
df -h
du -sh *

# 2. 清理
docker system prune -a
rm logs/*.log
find output -mtime +30 -delete
```
